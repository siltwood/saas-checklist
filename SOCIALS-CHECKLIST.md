# Socials & Brand Presence Checklist

Claiming, securing, and wiring up social accounts for a product launch.

## 1. Claim the name (do this the day you buy the domain)

- [ ] Handle availability checked and claimed on: X/Twitter, Instagram, TikTok, YouTube, LinkedIn (company page), Facebook page, Reddit (username + subreddit if relevant), GitHub org, Bluesky, Threads
- [ ] Same handle everywhere, or a documented fallback pattern (`productapp`, `getproduct`, `producthq`)
- [ ] Even for platforms you won't use: claim + park the handle (squatting protection)
- [ ] Product Hunt / Hacker News / Discord vanity URL — claim if a launch there is planned

## 2. Secure every account

- [ ] Unique password per platform, all in the password manager
- [ ] 2FA on every account — authenticator app or passkey, not SMS
- [ ] Recovery email points at an address you control long-term (ops account, not a personal one you might lose — see EMAIL-LOCKDOWN)
- [ ] Connected/third-party apps reviewed on each platform; revoke anything stale
- [ ] If more than one person posts: use platform team/role features, never shared passwords
- [ ] Recovery codes exported and stored offline

## 3. Profile completeness (consistent everywhere)

- [ ] Avatar: logo at each platform's size, legible at 32px
- [ ] Banner/cover image where supported
- [ ] Bio: one-liner value prop + link to site
- [ ] Link points at the marketing site with UTM params (`?utm_source=twitter&utm_medium=social`) so GA can attribute
- [ ] Pinned post: what the product is, one visual, one link
- [ ] Website ↔ social cross-links: footer icons on the site, site link on every profile

## 4. Content & launch wiring

- [ ] Share cards render correctly when your site is posted on each platform (see SHARE-CARDS checklist — test before launch day, cache is sticky)
- [ ] A launch post drafted per platform you actually plan to use (don't spray-and-abandon; two active channels beat six dead ones)
- [ ] Demo video/GIF assets exported in per-platform dimensions (16:9 YouTube/X, 9:16 TikTok/Reels/Shorts, 1:1 feed)
- [ ] Watermark/branding on shareable user-generated exports if your product produces media — every share is an ad
- [ ] Social listening minimal setup: notifications on mentions, saved search for product name + domain
- [ ] Support expectations set: if DMs are open, someone checks them; otherwise say where support lives

## 5. Ongoing hygiene

- [ ] Quarterly: re-audit 2FA, connected apps, and who has access
- [ ] Handles + credentials documented in the ops password manager, not one founder's head
